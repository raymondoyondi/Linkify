# Contributing to Linkify

First off, thank you for taking the time to contribute! 🎉 

Contributions from the community help make Linkify a better tool for everyone. Please read through these guidelines before getting started.

## Code of Conduct

By participating in this project, you agree to maintain a respectful, welcoming, and inclusive environment for all contributors.

## How Can I Contribute?

### Reporting Bugs
* Check the **Issues** tab to see if the bug has already been reported.
* If not, open a new issue using the bug report template.
* Include clear steps to reproduce the issue, the expected behavior, and screenshots if applicable.

### Suggesting Features
* Open a new issue and select the feature request template.
* Clearly describe the feature, why it would be useful, and how it fits into the Linkify tool.

### Submitting Pull Requests (PRs)
1. **Fork** the repository to your own GitHub account.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/raymondoyondi/Linkify.git
   ```
3. **Create a branch** for your changes using a descriptive name:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** and ensure your code follows the existing style guidelines.
5. **Commit your changes** with a clear and concise commit message:
   ```bash
   git commit -m "Add feature: custom theme colors for user bio"
   ```
6. **Push** the branch to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
7. Open a **Pull Request** against the `main` branch of the original Linkify repository.

## Development Setup

To set up Linkify on your local machine, follow these quick steps:

1. Install dependencies:
   ```bash
   npm install
   ```
2. Start the local development server:
   ```bash
   npm run dev
   ```
3. Run the test suite before submitting code:
   ```bash
   npm test
   ```

## Questions?

If you need help or have questions about the codebase, feel free to open a discussion on GitHub or comment directly on the relevant issue.
